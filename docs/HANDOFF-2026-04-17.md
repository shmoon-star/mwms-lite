## 4. 핵심 학습 / 다음에도 적용할 패턴

### 4-1. Supabase 페이지네이션 헬퍼 (재사용 가능)
`src/app/api/wms/asn/route.ts` 의 `fetchAllPaginated` + `fetchByIdsChunked` 패턴을 다른 라우트에서도 활용 가능.
- 단일 .order() 쿼리: `fetchAllPaginated`
- .in() with 많은 IDs: `fetchByIdsChunked` (50개 chunk)
- IDs 많고 결과도 많은 경우: 둘 다 결합 (chunk loop 안에서 paginate)

### 4-2. `??` vs `||` vs `> 0` 체크
- `??` (nullish): null/undefined만 fallback. **숫자 필드에 위험** (0이 막힘)
- `||` (truthy): 0/""/false도 fallback. 의도적이면 OK
- `> 0` 명시적 체크: 가장 안전, 의도 명확
- 권장: 숫자 필드에 fallback이 필요하면 명시적 `> 0` 체크

### 4-3. CSV 업로드 2단계 패턴 (Preview → Apply)
신규 Bulk Upload 기능들이 잘 따랐음:
- Preview: 파일 파싱 + 검증 + 충돌 체크 → 미리보기 반환
- Apply: valid 행만 받아서 실제 INSERT (서버 측 재검증으로 race 방지)
- 5가지 상태 분류로 작업자가 의사결정 가능
